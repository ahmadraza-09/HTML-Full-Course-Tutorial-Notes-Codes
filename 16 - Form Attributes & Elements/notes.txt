游리 HTML Form Attributes 

    This chapter describes the different attributes for the HTML <form> element.
    
游리 The Action Attribute

    游댖The action attribute defines the action to be performed when the form is submitted.
    游댖Usually, the form data is sent to a file on the server when the user 
    clicks on the submit button.
    游댖In the example below, the form data is sent to a file called 
    "action_page.php". This file contains a server-side script that 
    handles the form data:

    For example:

        <form action="data.html">
            <label for="username">Username</label><br>
            <input type="text" name="username"><br>
            <label for="password">Password</label><br>
            <input type="password" name="password"><br><br>
            <input type="submit" value="Submit">
        </form>

游리 The Target Attribute

    游댖The target attribute specifies where to display the response that is 
    received after submitting the form.
    游댖The target attribute can have one of the following values:

        _blank	    The response is displayed in a new window or tab
        _self	    The response is displayed in the current window
        _parent	    The response is displayed in the parent frame
        _top	    The response is displayed in the full body of the window

游리 The Method Attribute

    游댖The method attribute specifies the HTTP method to be used when 
    submitting the form data.
    游댖The form-data can be sent as URL variables (with method="get") or 
    as HTTP post transaction (with method="post").
    游댖The default HTTP method when submitting form data is GET. 
    
    For example:

        <form action="/action_page.php" method="get">
        <form action="/action_page.php" method="post">

        游리Notes on GET:

        游댖Appends the form data to the URL, in name/value pairs
        游댖NEVER use GET to send sensitive data! (the submitted form data 
        is visible in the URL!)
        游댖The length of a URL is limited (2048 characters)
        游댖Useful for form submissions where a user wants to bookmark the result
        游댖GET is good for non-secure data, like query strings in Google

        游리Notes on POST:

        游댖Appends the form data inside the body of the HTTP request (the 
        submitted form data is not shown in the URL)
        游댖POST has no size limitations, and can be used to send large amounts of data.
        游댖Form submissions with POST cannot be bookmarked

游리 The Autocomplete Attribute

    游댖The autocomplete attribute specifies whether a form should have 
    autocomplete on or off.
    游댖When autocomplete is on, the browser automatically complete values 
    based on values that the user has entered before. 

    For example:

        <form action="/action_page.php" autocomplete="on">


游리 HTML Form Elements

    This chapter describes all the different HTML form elements.

游리 The HTML <form> Elements
    
    The HTML <form> element can contain one or more of the following form elements:

        <input>
        <label>
        <select>
        <textarea>
        <button>
        <fieldset>
        <legend>
        <datalist>
        <output>
        <option>
        <optgroup>

游리 The <input> Element

    游댖One of the most used form elements is the <input> element.
    游댖The <input> element can be displayed in several ways, depending 
    on the type attribute.

    For example:    

        <label for="fname">First name:</label>
        <input type="text" id="fname" name="fname">

游리 The <label> Element

    The <label> element defines a label for several form elements.
    The <label> element is useful for screen-reader users, because the screen-reader will read out loud the label when the user focus on the input element.
    The <label> element also help users who have difficulty clicking on very small regions (such as radio buttons or checkboxes) - because when the user clicks the text within the <label> element, it toggles the radio button/checkbox.
    The for attribute of the <label> tag should be equal to the id attribute of the <input> element to bind them together.

游리 The <select> Element

    The <select> element defines a drop-down list:

    For example:

        <label for="cars">Choose a car:</label>
        <select id="cars" name="cars">
            <option value="volvo">Volvo</option>
            <option value="saab">Saab</option>
            <option value="fiat">Fiat</option>
            <option value="audi">Audi</option>
        </select>

        游댖The <option> element defines an option that can be selected.
        游댖By default, the first item in the drop-down list is selected.
        游댖To define a pre-selected option, add the selected attribute to the option:

游리 Visible Values:

Use the size attribute to specify the number of visible values:

For example;

    <label for="cars">Choose a car:</label>
    <select id="cars" name="cars" size="3">
        <option value="volvo">Volvo</option>
        <option value="saab">Saab</option>
        <option value="fiat">Fiat</option>
        <option value="audi">Audi</option>
    </select>

游리 Allow Multiple Selections:

    Use the multiple attribute to allow the user to select more than one value:

    For example:

        <label for="cars">Choose a car:</label>
            <select id="cars" name="cars" size="4" multiple>
            <option value="volvo">Volvo</option>
            <option value="saab">Saab</option>
            <option value="fiat">Fiat</option>
            <option value="audi">Audi</option>
        </select>

游리 The <textarea> Element

    The <textarea> element defines a multi-line input field (a text area):

    For example:

        <textarea name="message" rows="10" cols="30">
            The cat was playing in the garden.
        </textarea>

        The rows attribute specifies the visible number of lines in a text area.
        The cols attribute specifies the visible width of a text area.

游리 The <button> Element
    
    The <button> element defines a clickable button:

    For example:

        <button type="button" onclick="alert('Hello World!')">Click Me!</button>

游리 The <fieldset> and <legend> Elements

    游댖The <fieldset> element is used to group related data in a form.
    游댖The <legend> element defines a caption for the <fieldset> element.

    For example:

        <form action="/action_page.php">
            <fieldset>
                <legend>Personalia:</legend>
                <label for="fname">First name:</label><br>
                <input type="text" id="fname" name="fname" value="John"><br>
                <label for="lname">Last name:</label><br>
                <input type="text" id="lname" name="lname" value="Doe"><br><br>
                <input type="submit" value="Submit">
            </fieldset>
        </form>

游리 The <datalist> Element

    游댖The <datalist> element specifies a list of pre-defined options for 
    an <input> element.
    游댖Users will see a drop-down list of the pre-defined options as they input data.
    游댖The list attribute of the <input> element, must refer to the id attribute 
    of the <datalist> element.

    For example
        <form action="/action_page.php">
        <input list="browsers">
            <datalist id="browsers">
                <option value="Edge">
                <option value="Firefox">
                <option value="Chrome">
                <option value="Opera">
                <option value="Safari">
            </datalist>
        </form>
